1. 프로젝트 개요
- 대자연의 풍경을 보면 마음이 차분해지고 따뜻해지는 평화로운 배경화면의 매력을 알리고 싶어 이 사이트를 만들 게 되었습니다.
- 지금까지 배운 HTML, CSS 지식으로 혼자서 직접 사이트를 제작하는 실습하는 시간이 거의 없었기에 코딩 연습을 많이 할 수 없었습니다. 물론 집에서도 할 수 있지만, 정확한 목적을 가지고 있지 않으면 코딩할 마음이 생기지 않기에 이번 4일 동안 하는 실습시간을 한 번 제대로 활용해보자는 목적을 두었습니다. 

2. 화면 구성
- header 영역부터 만들어 section으로 main에 들어갈 본문을 순서대로 구성하였습니다.

3. HTML 작성 시 신경 쓴 점
- div를 구성할 때 부모태그, 자식태그을 뚜렷하게 구현할 수 있도록 구성하였습니다.

4. CSS 작성 시 신경 쓴 점
- CSS에서 가장 어려워했던 부분 애니메이션(transition, transform, animation)인데, 한 번도 제대로 구현해보지도 못하고 항상 실패하고 포기했었는데, 이번 사이트 구현 실습에는 포기 하지 않고, 애니메이션 부분을 구성할 수 있게 하였습니다. 퀄리티는 매우 낮지만 구현했다는 것만으로도 좋았습니다.
- 애니메이션 다음으로 제일 힘들었던 부분은 넓이/높이 설정, 글자 위치 등 각각 균형을 맞게 구현하는 게 너무 힘들었습니다. 이 부분 때문에 CSS 작업하는 데에 시간을 많이 써서 진행속도가 많이 느립니다. 완벽하지는 않지만 이번에 만든 사이트에서 최대한 각각 위치 맞도록 구성하였습니다.

5. 구현 중 문제와 해결
- background-image가 안 되는 문제가 있었습니다.
- width값과 height값을 정해놓으면 background-image가 잘 작용되어 문제를 해결할 수 있었습니다.
- 별 생각 없이 구현했던 width/height를 제대로 확인하지 않으면 background-image뿐만 아니라 다른 속성에서도 영향이 갈 수 있는 것을 알 수 있었습니다.


6. 실습을 통해 확인한 점

- display(block,inline, inline-block)부분을 이해하지 못했던 걸 이번 실습으로 알 수 있었습니다.
- CSS,HTML 작업할 때, 순서 맞게 코딩해야 나중에 다시 작업할 때, 꼬이지 않고 작업을 이어갈 수 있다는 것을 알게 되었습니다.
- 작업속도를 높일 수 있게 단축키를 활용하면 큰 도움이 되는 것을 느낄 수 있었습니다.

7. 개선해야할 점
- HTML/CSS 본격적인 작업에 들어가기 전, 아무생각 없이 시작해서 중간에 코드 꼬여버리는 상황이 자주 찾아와 스트레스를 받는 날이 많았는데, 코딩하기 전 어떤 주제로 만들건지 아이디어를 생각한 다음에 코딩 작업하는 것을 익혀야할 것 같습니다.

- 한 번 오류나면 쉽게 포기하고 싶다는 마음이 드는 걸 해결해야 할 것 같습니다. 이 때문에 작업속도가 매우 느려지고 사이트의 퀄리티도 급격히 떨어져 다른 사람들이 만든 사이트와 너무 비교되는 느낌이 들어, 코딩에 대한 자신감이 떨어지는 경험을 수 없이 겪었습니다. 코딩에 오류가 나도 괜히 짜쯩내지 않고 하기 싫다는 마음이 집어치우고 이 오류가 왜 났는지 알아보고 도저히 안되겠으면 GPT나 Gemini로 활용해서 해결할 수 있도록 인내력을 길러야 할 것 같습니다.
